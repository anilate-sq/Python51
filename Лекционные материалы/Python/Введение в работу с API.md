# Введение в работу с API

## Ключевые термины

**Application Programming Interface(API)** — инструмент, позволяющий разным приложениям обмениваться данными и функциями.

**HyperText Transfer Protocol(HTTP)** — протокол передачи информации в интернете для обмена запросами между клиентом и сервером

**Representation State Transfer(REST)** — архитектурный стиль построения API, использующий стандартные HTTP-методы

## Основные HTTP-запросы

- **GET** - запрос на получение
- **POST** - запрос на создание
- **PUT** - запрос на обновление существующих данных
- **DELETE** - запрос на удаление данных

## Основные HTTP-ответы

- **200 OK** — данные получены(GET) или обновлены(PUT)
- **201 CREATED** - данные были созданы/добавлены(POST)
- **400 Bad Request** - ошибка в запросе
- **401 Unauthorized** - API требует авторизацию
- **404 Not Found** - запрос на несуществующую страницу
- **500 Internal Server Error** - проблемы на стороне API
