# Виды конфликтов и способы разрешения


## Конфликты слияний

**Конфликт в git** - отсутствие возможности автоматического слияния.

**Причины возникновения конфликтов при слиянии веток:**
1. Изменения в одной и тойже строке, но в разных ветках
2. Изменения файла в одной ветка, когда он удален в другой
3. Два файла переименованы в одно имя 

**Конфликты с автоматическим разрешеним** - это конфликты возникаемые при работе с удаленным и локальным репозиторием.

**Виды конфликтов с автоматическим разрешением:**
1. Когда версия удаленного репозитория более поздняя и мы хотим сделать при этом git push
2. Когда версия локального репозитория более поздняя и мы хотим сделать при этом git pull

## Слияния

**Merge(слияние)** - это операция объеденения изменений из двух веток в одну.

**Виды слияний**
1. **Fast-Forward-Merge(Быстрое слияние)** - происходит, когда целевая ветка не изменялась с момента последнего коммита
2. **Three-Way-Merge(Трехстороннее слияние)** - происходит, когда обе ветки менялись после разветвления
